1. 前言概述

概要

本文档定义了接入CashGit印度唤醒支付系统的报文接口规范，具体包括：

  • 接口通信方式：涵盖网络通讯方式及接口加解密方式
  • 业务处理流程：涉及代收、代付、流水管理及财务管理等模块
  • 交易报文格式说明：包含入参、出参规范及JSON格式标准

目标人群

目标读者为具备开发能力、熟悉相关编程语言的开发、维护及管理人员。

💡 请求头说明：请求头Content-Type为：application/json;charset=utf-8，所有接口一致

相关术语

  • 平台或系统：指提供印度唤醒支付功能的我司系统，用于支持商户完成各类交易。
  • 商户号：这是商户在平台上的唯一标识，用于验证其在平台的合法性。
  • 商户订单号：商户系统为每笔订单生成的唯一编号，需确保其唯一性，以便在平台中进行区分。
  • 支付类型：指用来区分支付产品类别的字段，具体内容详见接口描述。
  • 加密密钥：商户用于交易请求前对数据进行加密和验签的关键工具。
  • 币种：用于区分货币类型的字段，详情参考接口描述部分。
  • 后台异步回调：当交易处理完成后，系统会主动通知商户网站并携带相关结果信息。
  • 商户：指接入平台并使用其支付服务的客户或机构。
注意：密钥必须妥善保管，一旦泄露应及时联系平台处理，以避免可能的损失。

2. 签名算法

签名规则

Step 1. 排序拼接

将非空参数值按照参数名ASCII码从小到大排序（字典序），使用URL键值对的格式（即key1=value1&key2=value2…）拼接成字符串(strString)。

  • 参数名ASCII码从小到大排序
  • 如果参数的值为空不参与签名
  • 参数名区分大小写

Step 2. 拼接key加密

将strString最后拼接上key得到strSign字符串，并对strSign进行SHA-256，得到sign值signValue(16进制小写)

示例参考【Java】

// 引入依赖
<dependency>
    <groupId>cn.hutool</groupId>
    <artifactId>hutool-core</artifactId>
    <version>5.8.26</version>
</dependency>

/**
 * 计算SHA-256摘要值，并转为16进制字符串
 */
public static void main(String[] args) {
    Map<String, Object> map = new TreeMap<>();
    map.put("mchNo", "M171925157713");
    map.put("mchOrderId", "1234567as94");
    map.put("timestamp", "1725300081000");
    map.put("payType", 102);
    map.put("notifyUrl", "http://localhost:8080/test");
    map.put("amount", "1000");
    
    String sign = checkSha256Hex(map, "123456789", "key");
    map.put("sign", sign);
    System.out.println(sign);
}

/**
 * 计算SHA-256摘要值，并转为16进制字符串
 * @param key   密钥（后台获取）
 * @param signWebkey 固定key
 */
public static String checkSha256Hex(Map<String, Object> obj, String key, String signWebKey) {
    TreeMap<String, Object> map = mapToTreeMap(obj);
    String data = strSign(map, key, signWebKey);
    System.out.println(data);
    return sha256Hex(data);
}

public static String sha256Hex(String data) {
    return DigestUtil.sha256Hex(data);
}

/**
 * 字符拼接k=1&v=2形式
 * @param map 对象
 * @param key 密钥
 * @param signKye 签名key
 * @return 返回拼接字符串
 */
public static String strSign(TreeMap<String, Object> map, String key, String signKye) {
    StringBuilder builder = new StringBuilder();
    map.forEach((k, v) -> {
        builder.append(k).append("=").append(v).append("&");
    });
    builder.append(signKye).append("=").append(key);
    return builder.toString();
}

/**
 * map转TreeMap 忽略空值
 * @param map map
 * @return 返回TreeMap
 */
public static TreeMap<String, Object> mapToTreeMap(Map<String, Object> map) {
    TreeMap<String, Object> treeMap = new TreeMap<>(map);
    // 使用迭代器来安全地移除元素
    treeMap.entrySet().removeIf(entry -> entry.getValue() == null ||
            StrUtil.isEmpty(entry.getValue().toString()));
    return treeMap;
}

验签

验签是验证接收到的数据完整性和真实性的重要步骤。商户在收到异步通知或查询结果后，必须进行验签验证。

验签步骤

  1. 接收数据后，提取除sign字段外的所有参数
  2. 按照签名算法重新计算签名
  3. 将计算出的签名与接收到的sign字段进行对比
  4. 如果一致，则数据完整且来源可信
重要提醒：验签失败的数据绝对不能用于业务处理，应立即丢弃并记录日志。

4. 出款接口

出款接口用于向用户银行账户转账，支持多种银行和账户类型。系统支持印度主要银行的IMPS、NEFT、RTGS等转账方式。

支持的银行类型

银行类型 说明 状态
公共部门银行 SBI、PNB、BOB、Canara Bank等 已支持
私营银行 HDFC、ICICI、Axis Bank、Kotak等 已支持
外资银行 Citibank、Standard Chartered等 已支持
合作银行 各类合作银行 部分支持

转账方式

  • IMPS (Immediate Payment Service)：实时转账，24x7可用，限额20万卢比
  • NEFT (National Electronic Funds Transfer)：批量转账，工作日处理，无金额限制
  • RTGS (Real Time Gross Settlement)：大额实时转账，工作日处理，最低2万卢比

  • 可用金额：可以用于新交易的金额
  • 余额查询不收取任何费用
  • 建议定期查询余额，确保账户资金充足

错误码 说明 解决方案 严重程度

9998 服务维护中 等待服务恢复 中等
9997 网络超时 检查网络连接，重试请求 中等
