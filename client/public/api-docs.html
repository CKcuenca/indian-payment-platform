1. Sign加密

所有API请求都需要进行MD5签名验证，确保数据安全和完整性。

签名规则

1. 将所有参数按照参数名ASCII码从小到大排序 2. 拼接成 key1=value1&key2=value2 格式 3. 在字符串末尾拼接上商户密钥 4. 对拼接后的字符串进行MD5加密（32位小写）

签名示例

// 原始参数 mchid=MERCHANT_001 amount=100.00 out_trade_no=ORD1234567890 notify_url=https://yourgame.com/payment/notify // 排序后拼接 amount=100.00&mchid=MERCHANT_001¬ify_url=https://yourgame.com/payment/notify&out_trade_no=ORD1234567890 // 拼接密钥 amount=100.00&mchid=MERCHANT_001¬ify_url=https://yourgame.com/payment/notify&out_trade_no=ORD1234567890YOUR_SECRET_KEY // MD5加密结果 sign=md5("amount=100.00&mchid=MERCHANT_001¬ify_url=https://yourgame.com/payment/notify&out_trade_no=ORD1234567890YOUR_SECRET_KEY")

注意事项

重要提示：
  • 签名验证失败会导致API调用失败
  • 请确保签名算法正确实现
  • 参数值不能为空，空值不参与签名
  • 签名不区分大小写，但建议使用小写

2.1 创建代收订单

为玩家创建充值订单，支持UPI支付方式。

接口地址： POST https://api.merchant.cashgit.com/api/collection/create

请求参数

参数名 类型 必填 说明
mchid string 是 商户ID
pay_id string 是 支付ID
out_trade_no string 是 商户订单号，唯一标识
amount string 是 订单金额（保留2位小数）
notify_url string 否 异步通知URL
sign string 是 MD5签名

请求示例

{ "mchid": "MERCHANT_001", "pay_id": "PAY_001", "out_trade_no": "ORD1234567890", "amount": "100.00", "notify_url": "https://yourgame.com/payment/notify" }

响应示例

{ "rCode": 200, "message": "订单创建成功", "data": { "trade_no": "TXN_123456", "out_trade_no": "ORD1234567890", "amount": "100.00", "status": "0", "qr_code": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...", "payment_url": "https://pay.cashgit.com/pay/TXN_123456" } }
注意： 订单创建成功后，玩家可以通过UPI应用扫描二维码或使用支付链接完成支付。

2.2 代收回调

支付成功后，平台会向商户指定的notify_url发送异步通知。

回调参数

参数名 类型 说明
mchid string 商户ID
out_trade_no string 商户订单号
trade_no string 平台交易号
amount string 订单金额
status string 订单状态
utr_number string UTR交易号
sign string MD5签名

回调示例

POST https://yourgame.com/payment/notify Content-Type: application/x-www-form-urlencoded mchid=MERCHANT_001&out_trade_no=ORD1234567890&trade_no=TXN_123456&amount=100.00&status=2&utr_number=UTR123456789&sign=abc123def456

回调处理要求

重要： 商户收到回调后，必须返回"SUCCESS"字符串，否则平台会重复发送回调通知。

回调重试机制

回调重试时间间隔： - 第1次：立即发送 - 第2次：5分钟后重试 - 第3次：15分钟后重试 - 第4次：30分钟后重试 - 最多重试4次，如果仍然失败则停止

2.3 查询代收订单状态

查询订单的当前状态和详细信息。

接口地址： POST https://api.merchant.cashgit.com/api/collection/query

请求参数

参数名 类型 必填 说明
mchid string 是 商户ID
out_trade_no string 是 商户订单号
sign string 是 MD5签名

响应示例

{ "rCode": 200, "message": "查询成功", "data": { "out_trade_no": "ORD1234567890", "trade_no": "TXN_123456", "amount": "100.00", "status": "2", "create_time": "2024-01-15 10:00:00", "update_time": "2024-01-15 10:30:00", "utr_number": "UTR123456789" } }

订单状态说明

状态码说明： 0 - 待处理 (PENDING) 1 - 处理中 (PROCESSING) 2 - 成功 (SUCCESS) 3 - 失败 (FAILED) 4 - 取消 (CANCELLED) 5 - 过期 (EXPIRED)

2.4 UTR补单

当玩家通过UPI支付后，可以通过UTR号进行补单操作。

接口地址： POST https://api.merchant.cashgit.com/api/utr/supplement

请求参数

参数名 类型 必填 说明
mchid string 是 商户ID
utr_number string 是 UTR交易号
amount string 是 交易金额
sign string 是 MD5签名

UTR状态查询

接口地址： POST https://api.merchant.cashgit.com/api/utr/query
// UTR状态查询参数 { "mchid": "MERCHANT_001", "utr_number": "UTR123456789", "sign": "abc123def456" } // 响应示例 { "rCode": 200, "message": "查询成功", "data": { "utr_number": "UTR123456789", "status": "SUCCESS", "amount": "100.00", "transaction_time": "2024-01-15 10:30:00" } }

3.1 创建代付订单

为玩家创建提现订单，将资金转入指定银行账户。

接口地址： POST https://api.merchant.cashgit.com/api/payout/create

请求参数

参数名 类型 必填 说明
mchid string 是 商户ID
pay_id string 是 支付ID
out_trade_no string 是 商户订单号
amount string 是 提现金额（保留2位小数）
account_number string 是 银行账户号
ifsc_code string 是 IFSC代码
account_holder string 是 账户持有人姓名
notify_url string 否 异步通知URL
sign string 是 MD5签名

请求示例

{ "mchid": "MERCHANT_001", "pay_id": "PAY_001", "out_trade_no": "Payout_1234567890", "amount": "500.00", "account_number": "1234567890", "ifsc_code": "SBIN0001234", "account_holder": "Player Name", "notify_url": "https://yourgame.com/payout/notify" }

响应示例

{ "rCode": 200, "message": "代付订单创建成功", "data": { "trade_no": "TXN_789012", "out_trade_no": "Payout_1234567890", "amount": "500.00", "status": "0" } }

3.2 代付回调

代付处理完成后，平台会向商户指定的notify_url发送异步通知。

回调参数

参数名 类型 说明
mchid string 商户ID
out_trade_no string 商户订单号
trade_no string 平台交易号
amount string 代付金额
status string 代付状态

回调示例

POST https://yourgame.com/payout/notify Content-Type: application/x-www-form-urlencoded mchid=MERCHANT_001&out_trade_no=Payout_1234567890&trade_no=TXN_789012&amount=500.00&status=2&sign=abc123def456

代付状态说明

状态码说明： 0 - 待处理 (PENDING) 1 - 处理中 (PROCESSING) 2 - 成功 (SUCCESS) 3 - 失败 (FAILED) 4 - 取消 (CANCELLED) 5 - 拒绝 (REJECTED)
