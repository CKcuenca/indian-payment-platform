PassPay Merchant API 对接文档

专业的印度支付平台，支持UPI充值、银行代付、UTR验证等功能

请输入访问密码

1. 创建代收订单

为玩家创建充值订单，支持UPI支付方式

接口地址： POST https://api.merchant.passpay.cc/api/developer/order/create

请求参数

参数名 类型 必填 说明
mchid string 是 商户ID
pay_id string 是 支付ID
out_trade_no string 是 商户订单号，唯一标识
amount string 是 订单金额（保留2位小数）
notify_url string 否 异步通知URL
sign string 是 MD5签名

请求示例

{ "mchid": "MERCHANT_001", "pay_id": "PAY_001", "out_trade_no": "ORD1234567890", "amount": "100.00", "notify_url": "https://yourgame.com/payment/notify" }

响应示例

{ "rCode": 200, "message": "订单创建成功", "data": { "trade_no": "TXN_123456", "out_trade_no": "ORD1234567890", "amount": "100.00", "status": "0" } }
注意： 订单创建成功后，玩家可以通过UPI应用扫描二维码或使用支付链接完成支付。

2. 查询代收订单状态

查询订单的当前状态和详细信息

接口地址： POST https://api.merchant.passpay.cc/api/developer/order/query

请求参数

参数名 类型 必填 说明
mchid string 是 商户ID
pay_id string 是 支付ID
out_trade_no string 是 商户订单号
trade_no string 是 平台交易号
sign string 是 MD5签名

订单状态说明

状态码说明： 0 - 待处理 (PENDING) 1 - 处理中 (PROCESSING) 2 - 成功 (SUCCESS) 3 - 失败 (FAILED) 4 - 取消 (CANCELLED) 5 - 过期 (EXPIRED)

1. 创建代付订单

为玩家创建提现订单，将资金转入指定银行账户

接口地址： POST https://api.merchant.passpay.cc/api/developer/payout/create

请求参数

参数名 类型 必填 说明
mchid string 是 商户ID
pay_id string 是 支付ID
out_trade_no string 是 商户订单号
amount string 是 提现金额（保留2位小数）
account_number string 是 银行账户号
ifsc_code string 是 IFSC代码
account_holder string 是 账户持有人姓名
notify_url string 否 异步通知URL
sign string 是 MD5签名

请求示例

{ "mchid": "MERCHANT_001", "pay_id": "PAY_001", "out_trade_no": "Payout_1234567890", "amount": "500.00", "account_number": "1234567890", "ifsc_code": "SBIN0001234", "account_holder": "Player Name", "notify_url": "https://yourgame.com/payout/notify" }

代付状态说明

状态码说明： 0 - 待处理 (PENDING) 1 - 处理中 (PROCESSING) 2 - 成功 (SUCCESS) 3 - 失败 (FAILED) 4 - 取消 (CANCELLED) 5 - 拒绝 (REJECTED)

1. 异步回调通知

当订单状态发生变化时，系统会向您配置的notify_url发送POST请求

重要： 回调通知采用重试机制，如果您的服务器返回非200状态码，系统会在5分钟、15分钟、30分钟后重试，最多重试3次。

回调请求格式

POST /your-notify-url Content-Type: application/json { "orderId": "ORD1234567890", "tradeNo": "TXN_123456", "status": "2", "amount": "100.00", "utr": "UTR123456789", "timestamp": "2024-01-15T10:05:00Z" }

回调响应要求

您的服务器收到回调后，必须返回HTTP 200状态码和以下JSON格式：

{ "success": true, "message": "received" }

2. 签名验证

所有API请求都需要进行MD5签名验证，确保数据安全

MD5签名生成步骤：

  1. 参数排序： 所有请求参数按ASCII码从小到大排序
  2. 拼接参数： 按 key=value&key=value... 格式拼接参数签名源串
  3. 添加密钥： 拼接好的源串最后拼接上 secret key
  4. 计算MD5： 计算最终拼接好签名源串的MD5散列值
  5. 提交签名： 计算得到的散列值作为sign参数的值

签名示例

假设请求参数为: mchid=MERCHANT_001&pay_id=PAY_001&out_trade_no=ORD123&amount=100.00 排序后: amount=100.00&mchid=MERCHANT_001&out_trade_no=ORD123&pay_id=PAY_001 添加密钥: amount=100.00&mchid=MERCHANT_001&out_trade_no=ORD123&pay_id=PAY_001&key=your_secret_key MD5签名: {计算得到的MD5值}
