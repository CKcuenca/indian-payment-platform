CashGit 商户API对接文档

专业的印度支付平台，支持UPI充值、银行代付等功能

请输入访问密码

1. 创建代收订单

为玩家创建充值订单，返回支付链接

接口地址： POST /api/payment/create

请求参数

参数名 类型 必填 说明
merchantId string 是 商户ID
amount integer 是 充值金额（分）
currency string 否 货币代码，默认INR
customerEmail string 是 玩家邮箱
customerPhone string 是 玩家手机号
returnUrl string 是 支付完成后的跳转URL
notifyUrl string 否 异步通知URL

请求示例

{ "merchantId": "MERCHANT_001", "amount": 10000, "currency": "INR", "customerEmail": "player@example.com", "customerPhone": "919876543210", "returnUrl": "https://yourgame.com/payment/return", "notifyUrl": "https://yourgame.com/payment/notify" }

响应示例

{ "success": true, "data": { "orderId": "ORD1234567890", "paymentUrl": "https://cashgit.com/pay/ORD1234567890", "qrCode": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...", "expiresAt": "2024-01-15T11:00:00Z" } }

2. 创建代付订单

为玩家创建提现订单，将资金转入指定银行账户

接口地址： POST /api/payment/withdraw

请求参数

参数名 类型 必填 说明
merchantId string 是 商户ID
amount integer 是 提现金额（分）
bankAccount object 是 银行账户信息
customerEmail string 是 玩家邮箱
customerPhone string 是 玩家手机号

银行账户信息结构

{ "accountNumber": "1234567890", "ifscCode": "SBIN0001234", "accountHolderName": "Player Name", "bankName": "State Bank of India" }

请求示例

{ "merchantId": "MERCHANT_001", "amount": 50000, "currency": "INR", "bankAccount": { "accountNumber": "1234567890", "ifscCode": "SBIN0001234", "accountHolderName": "Player Name", "bankName": "State Bank of India" }, "customerEmail": "player@example.com", "customerPhone": "919876543210" }

3. 查询订单状态

查询订单的当前状态和详细信息

接口地址： GET /api/payment/status/:orderId

响应示例

{ "success": true, "data": { "orderId": "ORD1234567890", "merchantId": "MERCHANT_001", "type": "DEPOSIT", "amount": 10000, "currency": "INR", "status": "SUCCESS", "provider": { "name": "passpay", "transactionId": "TXN_123456" }, "createdAt": "2024-01-15T10:00:00Z", "updatedAt": "2024-01-15T10:05:00Z" } }

4. 查询商户余额

查询商户账户的当前余额和额度信息

接口地址： GET /api/merchant/balance

响应示例

{ "success": true, "data": { "merchantId": "MERCHANT_001", "balance": 1500000, "currency": "INR", "limits": { "dailyLimit": 10000000, "monthlyLimit": 100000000, "singleTransactionLimit": 5000000 }, "usage": { "dailyUsed": 2500000, "monthlyUsed": 15000000, "lastResetDate": "2024-01-01" } } }

5. 异步回调通知

当订单状态发生变化时，系统会向您配置的notifyUrl发送POST请求

回调通知采用重试机制，如果您的服务器返回非200状态码，系统会在5分钟、15分钟、30分钟后重试，最多重试3次。

回调请求格式

POST /your-notify-url Content-Type: application/json X-Signature: signature_here { "orderId": "ORD1234567890", "merchantId": "MERCHANT_001", "status": "SUCCESS", "amount": 10000, "currency": "INR", "providerTransactionId": "TXN_123456", "timestamp": "2024-01-15T10:05:00Z", "signature": "calculated_signature" }

回调响应要求

您的服务器收到回调后，必须返回HTTP 200状态码和以下JSON格式：

{ "success": true, "message": "received" }
